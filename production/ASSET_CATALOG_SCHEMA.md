# Asset Catalog Schema

Use this schema across Airtable, Notion, Google Drive, and GitHub.

## Required Fields

| Field | Purpose |
|---|---|
| Asset ID | Canonical ID used across platforms |
| Title | Human-readable title |
| Asset Type | Cover, ecosystem diagram, carousel, prompt, audio, PDF, workbook, manuscript |
| Series | Major Moves Series or related content family |
| Volume | Book or campaign volume number |
| Status | Inbox, Cataloged, In Review, Approved, Published, Archived |
| Source Filename | Original uploaded file name |
| GitHub Path | Canonical repo location |
| Google Drive Location | Asset vault folder or file link |
| Notion Page | Knowledge library page |
| Airtable Record | Production tracker record |
| Platform Role | Cover, doctrine seed, teaching asset, sales asset, social asset, internal OS |
| Related Pillars | Major AI Mindset pillars connected to the asset |
| Notes | Context, usage ideas, or production notes |

## Asset ID Format

MMS-[TYPE]-[SHORTNAME]-V001

Examples:

- MMS-COVER-YOU-ARE-THE-SYSTEM-V001
- MMS-DIAGRAM-CONTENT-ECOSYSTEM-V001
- MMS-AUDIO-BUSINESS-EMPIRE-ONE-IMAGE-V001

## Catalog Rule

Every image must be cataloged before it becomes a book, carousel, PDF, or Gumroad product.
